🎨

RGB Colour Model

Three values — Red, Green, Blue — each 0 to 255. Mix them to make any colour. R255 G255 B0 = Yellow. All zeros = lights off.

🌬️

Breathing Lamp

A variable brightness counts 0 → 255 then back down. The LED fades in and out — like an inhale and exhale on repeat.

🔁

Repeat Until Loop

repeat until brightness > 255 — the loop keeps going until the condition is true. Change by 5 = fast fade. Change by 1 = smooth.

🌈

Colour Gradient

R drops as G rises — Red → Yellow → Green. Two variables moving in opposite directions create a smooth colour shift.

Task 01

Learn the three screen display methods

Little M has a 1.44-inch TFT screen (128×128 px). There are three ways to put content on it — know the difference before class.

Every pixel on Little M's TFT screen is individually controlled. In mBlock you choose how to lay out content:

  • /Screen Printing — prints text line by line, wraps at the edge. Content accumulates — use clear screen before reprinting.
  • /Label Display — pins text to a specific XY position or named corner. Same label number = overwrites in place. No clear needed.
  • /Table Display — up to 4 rows × 3 columns. Enter content cell by cell. Great for structured data like scores.
Think before classYou want to show a timer that updates every second. Which method — Screen Printing or Label Display? Why?

Task 03

Memorise the musical note numbers

Little M plays notes using MIDI numbers. The piano block covers C to B — memorise these seven before class.

C (Do) = 60 D (Re) = 62 E (Mi) = 64 F (Fa) = 65 G (So) = 67 A (La) = 69 B (Ti) = 71

Beat = duration: 0.25 beat = 0.25s · 0.5 beat = 0.5s · 1 beat = 1s · 2 beats = 2s. Use ← → arrows to change octave.

  • /Quick test — without looking: what number is G? What number is C?
  • /Notice: C→D is +2, but E→F is only +1. Why the uneven gaps?
Think before classDo Re Mi Fa Sol = 60, 62, 64, 65, 67. What would La (A) be? Figure it out from the pattern.
