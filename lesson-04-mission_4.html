1
Learn the RGB colour combinations

Every colour the mBot2 displays is built from three values: Red, Green, Blue — each from 0 (off) to 255 (full brightness). Mix them to make anything.

No app needed. Just memorise these before class:

R 255 G 0 B 0 → Red R 0 G 255 B 0 → Green R 0 G 0 B 255 → Blue
R 255 G 255 B 0 → Yellow R 0 G 255 B 255 → Cyan R 255 G 0 B 255 → Magenta R 255 G 255 B 255 → White

Quick test before class — without looking: what makes yellow? What makes white?

All values at 0 = black (lights off). The higher the number, the brighter that channel.
2
Spot a breathing light in real life

A breathing lamp gradually brightens and dims on repeat — like inhaling and exhaling. It's everywhere once you notice it.

Find one example. Some easy ones to get you started:

/ MacBook sleep
The white light slowly pulses when sleeping. The original breathing lamp.
/ Phone notification
Some Android phones pulse a colour when you have unread messages.
/ Gaming gear
RGB keyboards and mice often breathe slowly by default.
You'll share your example in the check-in form at the end. Bonus points for something unexpected.
3
Read this loop — predict what it does

In class we'll use a variable and a repeat until loop to fade the LED smoothly. Here's a simplified version — read it and think through what happens:

set brightness to 0
start dark
repeat until brightness > 255
keep going…
LED all displays R brightness G 0 B 0
wait 0.02 seconds
change brightness by 5
get brighter

Think through these before class — no need to write anything:

①
What colour does the LED start as? What does it look like at the end of the loop?
②
Why brightness > 255 instead of brightness = 255? What breaks if you use equals?
③
If you changed by 5 to by 1 — smoother or rougher? Why?
Variables are labelled boxes. "brightness" holds a number that changes each loop — that number drives the LED brightness directly.
